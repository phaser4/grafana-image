# Workspace Instructions

For every request in this repository:

1. Read `Spec.md`, `Technical Details.md`, and `Other.md` before starting work.
2. Use those files as the current project memory and source of truth.
3. After finishing the request, update those files to reflect any new requirements, implementation details, decisions, scope changes, or open questions introduced by the work.

If a request does not change project knowledge, those files should still be checked before work begins.
